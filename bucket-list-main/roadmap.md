# 개발자 웹 이력서 개발 로드맵

> **기술 스택:** HTML · CSS · JavaScript · Tailwind CSS
> **목표:** 빌드 도구 없이 정적 파일만으로 동작하는, 반응형·다크모드를 지원하는 개발자 웹 이력서(포트폴리오) 제작

---

## 0. 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| 형태 | 단일 페이지(One-page) 또는 섹션 스크롤형 웹 이력서 |
| 핵심 가치 | 빠른 로딩, 모바일 대응, 깔끔한 디자인, 쉬운 유지보수 |
| 배포 | GitHub Pages / Netlify / Vercel (정적 호스팅) |
| 도구 | 코드 에디터(VS Code), 브라우저 개발자 도구, Git |

---

## 1. 기획 단계 (1~2일)

- [ ] **목적 정의**: 취업용 / 프리랜스 홍보용 / 개인 브랜딩용인지 명확히 하기
- [ ] **타깃 독자**: 채용 담당자, 동료 개발자 등 누가 볼지 결정
- [ ] **담을 콘텐츠 목록화**
  - 자기소개(한 줄 소개 + 상세 소개)
  - 보유 기술(Skills)
  - 경력 / 프로젝트
  - 학력 / 자격증
  - 연락처 / SNS / GitHub 링크
- [ ] **레퍼런스 수집**: 마음에 드는 이력서·포트폴리오 3~5개 분석
- [ ] **와이어프레임 스케치**: 종이나 Figma로 섹션 배치 구상

### 추천 페이지 구조
```
┌─────────────────────────────┐
│  Header / Navigation        │
├─────────────────────────────┤
│  Hero (이름, 직무, 한 줄 소개) │
│  About (자기소개)            │
│  Skills (기술 스택)          │
│  Experience / Projects       │
│  Education                   │
│  Contact                     │
├─────────────────────────────┤
│  Footer                      │
└─────────────────────────────┘
```

## 2. 환경 설정 (0.5일)

- [ ] 프로젝트 폴더 및 파일 구조 생성
  ```
  resume/
  ├── index.html
  ├── css/
  │   └── styles.css      # Tailwind로 표현 못 하는 커스텀 스타일
  ├── js/
  │   └── main.js         # 인터랙션 로직
  ├── assets/
  │   ├── images/         # 프로필 사진, 프로젝트 썸네일
  │   └── resume.pdf      # 다운로드용 PDF 이력서(선택)
  └── README.md
  ```
- [ ] **Tailwind 적용 방식 결정**
  - **간단(추천 초안):** CDN 방식 — `<script src="https://cdn.tailwindcss.com"></script>`
  - **권장(실서비스):** Tailwind CLI로 빌드 → 사용하지 않는 클래스 제거(purge)로 용량 최적화
- [ ] Git 초기화 및 첫 커밋

---

## 3. 마크업 구조 (HTML) (1~2일)

- [ ] `index.html` 시맨틱 태그로 골격 작성 (`<header>`, `<main>`, `<section>`, `<footer>`)
- [ ] 각 섹션에 `id` 부여 (네비게이션 앵커 이동용: `#about`, `#skills` 등)
- [ ] 메타 태그 설정
  - [ ] `<meta name="viewport">` (모바일 필수)
  - [ ] `<title>`, `<meta name="description">`
  - [ ] **Open Graph 태그**(공유 시 미리보기): `og:title`, `og:description`, `og:image`
- [ ] **접근성(a11y)** 기본 적용: `alt` 속성, 적절한 heading 계층(h1→h2→h3), `aria-label`

---

## 4. 스타일링 (Tailwind + CSS) (2~3일)

- [ ] **레이아웃**: Flexbox / Grid 유틸리티로 섹션 배치
- [ ] **반응형 디자인**: `sm:` `md:` `lg:` 브레이크포인트로 모바일~데스크톱 대응
- [ ] **색상/타이포그래피**: 일관된 색상 팔레트와 폰트 위계 설정
- [ ] **다크모드**: Tailwind `dark:` 변형 또는 `prefers-color-scheme` 활용
- [ ] **컴포넌트 스타일링**
  - [ ] 네비게이션 바 (스크롤 시 고정 / 그림자)
  - [ ] Hero 섹션 (그라데이션 배경, CTA 버튼)
  - [ ] Skill 배지 / 프로그레스 바
  - [ ] 프로젝트 카드 (이미지 + 설명 + 링크)
- [ ] **커스텀 CSS**: Tailwind로 어려운 부분만 `styles.css`에 작성 (애니메이션 keyframe 등)

---

## 5. 인터랙션 (JavaScript) (1~2일)

- [ ] **부드러운 스크롤**: 네비게이션 클릭 시 해당 섹션으로 이동
- [ ] **모바일 햄버거 메뉴** 토글
- [ ] **다크모드 토글 버튼** + `localStorage`로 설정 저장
- [ ] **스크롤 애니메이션**: `IntersectionObserver`로 섹션 등장 효과
- [ ] **현재 섹션 하이라이트**: 스크롤 위치에 따라 활성 메뉴 표시(scrollspy)
- [ ] (선택) **타이핑 효과**, **Back to Top 버튼**, **연락처 폼 검증**

## 7. 최적화 & 품질 점검 (1일)

- [ ] **성능**: 이미지 압축, 불필요한 Tailwind 클래스 purge, lazy loading
- [ ] **SEO**: 메타 태그, 시맨틱 마크업, `robots.txt`, `sitemap.xml`(선택)
- [ ] **접근성**: 키보드 탐색, 색 대비, 스크린리더 테스트
- [ ] **크로스 브라우저 테스트**: Chrome / Safari / Firefox / 모바일
- [ ] **반응형 점검**: 다양한 화면 크기에서 깨짐 확인
- [ ] **Lighthouse 점수 측정** (Performance / Accessibility / SEO 90+ 목표)
